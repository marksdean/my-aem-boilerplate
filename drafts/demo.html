What is Edge Delivery Services?

Edge Delivery Services is a composable set of services that enables a high degree of flexibility in how you author and deliver content on the web. It allows for fast development cycles and fast websites, all measurable through Core Web Vitals.

How do blocks work?

Blocks are the reusable building blocks of AEM. Each block has a JavaScript file that decorates the initial HTML generated from authored content, and a CSS file that styles it. Blocks are self-contained and automatically loaded only when present on the page.

What is the three-phase loading model?

Pages load in three phases to maximize performance: Eager loads only what is needed for LCP (Largest Contentful Paint), Lazy loads the remaining page content including header and footer, and Delayed loads non-critical third-party scripts and analytics.

Overview

Project Overview

This project is built on the AEM boilerplate, providing a fast, accessible, and easy-to-author website foundation. It uses Edge Delivery Services to serve content at the edge for maximum performance.

Features

Key Features

  • Vanilla JavaScript — no framework overhead
  • Automatic image optimization
  • Core Web Vitals targeting 100 score
  • Content authored in Google Docs or SharePoint

Getting Started

Getting Started

Install dependencies with npm install, then run the dev server with aem up. Your site will be available at http://localhost:3000 with live reload.

The form block fetches field definitions from a JSON endpoint and renders a complete accessible form. It requires a .json data source:

Note: this requires a /forms/contact.json sheet to be authored and previewed with field definitions.
